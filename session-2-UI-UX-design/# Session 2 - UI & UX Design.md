## APPLICATION:
### Part A – Plan
Using the specification in appendix 1*, plan out the design and implementation of a mobile application that meets the specified requirements.
**The UX design should:**
1. Satisfy all functional requirements set out in appendix 1.
2. Identify necessary user inputs required for functionality.
3. Highlight any outputs and data displayed using appropriate controls.
4. Identify any events required that form the user interaction such as gestures (Click, Scroll, Swipe etc).
5. Be readable and accessible.
6. Maintain a clear, consistent design throughout.
Designs must be submitted and signed off by lecturer before work commences on the implementation (Part C).
**Deliverables:**
- Wireframes/UI Designs for each screen with consideration for orientation. (Can be done hard-copy or in design software of choice. Scan and attach any sketches done to the final submission.)
- Actions and Events list (if not included in designs).
- Requirements and feature list documentation. (Additional Template materials available)
- Stakeholder Sign-off completed.

> Design idea for the mobile application is a book tracking application and tournament style picker. 20 books will be selected based on the genre of choosing, then a pick the best tournament will happen where two book covers show up, you chose the one you like the most and then this is repeated until one book remains. This is you recommended book. You can then track your reading, earn a daily streak and get rewards.
